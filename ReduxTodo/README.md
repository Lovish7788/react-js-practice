# 🧠 Redux Toolkit (RTK) - Complete Master Notes

This project is a deep dive into **Global State Management** using **Redux Toolkit (RTK)**.  
It demonstrates how to move away from **Prop Drilling** and **Context API** toward a more scalable, industry-standard architecture.

---

## 📌 What is Redux?

Redux is a **pattern and library** for managing and updating application state using events called **actions**.  
It acts as a **centralized store** for state that needs to be accessed across your entire application.

---

## 🏦 The "Bank" Analogy

Understanding Redux becomes easier with this analogy:

- **The Store (The Vault)** → Where all your money (data) is kept safely.
- **The Action (The Deposit Slip)** → A note describing what you want to do (e.g., "Add 100 rupees").
- **The Reducer (The Teller)** → The person who processes your request and updates the vault.
- **Dispatch (The Delivery)** → The act of submitting the slip to the teller.

---

# 🛠️ Redux Toolkit (RTK) Workflow

Redux Toolkit is the **modern way to write Redux**.  
It removes boilerplate and makes state management predictable and clean.

---

## 1️⃣ The Store (`store.js`)

The **Single Source of Truth**.  
All application state lives here.

```javascript
import { configureStore } from "@reduxjs/toolkit";
import todoReducer from "../features/todo/todoSlice";

export const store = configureStore({
    reducer: todoReducer // Tells the store to use this logic
});
---
2️⃣ The Slice (todoSlice.js)

A Slice contains:

Initial State → The starting data (e.g., todos: [])

Reducers → Functions that define how state changes

Actions → Automatically generated by RTK
---
🧪 Immer (Under the Hood)

Redux Toolkit uses Immer, allowing you to write code like:

state.todos.push(newTodo)

Even though it looks mutable, it safely updates state immutably behind the scenes.

## Hook	Purpose	Example
useSelector	Read data from store	`const todos = useSelector(state => state.todos)`
useDispatch	Send action to store	`const dispatch = useDispatch()`
---
# ➕ Adding a Todo (AddTodo.jsx)

`const dispatch = useDispatch();

const addTodoHandler = (e) => {
    e.preventDefault();
    dispatch(addTodo(input)); // Sending input as payload
    setInput('');
}`
---

##📋 Reading & Deleting Todos (Todos.jsx)

We subscribe to the global state using useSelector and dispatch delete actions using the todo ID.

`const todos = useSelector(state => state.todos);
const dispatch = useDispatch();

{todos.map((todo) => (
    <li key={todo.id}>
        {todo.text}
        <button onClick={() => dispatch(removeTodo(todo.id))}>
            Delete
        </button>
    </li>
))}`
